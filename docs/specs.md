# Specs：夸克PC桌面端Mini知识助手

## 1. 功能规格（In-Scope）

### F1：文件浏览面板
**描述**：左侧面板以树形结构展示用户选择的本地目录结构，支持文件夹展开/收起和文件类型图标。

**具体行为**：
- 启动时左侧面板显示 "选择文件夹" 按钮，点击弹出系统目录选择对话框
- 用户选择目录后，主进程递归读取目录结构（限制深度2层、最多100个文件/文件夹）
- 文本文件（.txt .md .json .js .ts .py .csv）读取前200行内容作为 `content` 字段
- 非文本文件（.pdf .png .jpg .xlsx .docx 等）只读取元信息（名称、大小、修改时间、扩展名），不读取内容
- 点击文件夹节点 → 展开/收起子节点
- 点击文件节点 → 高亮选中，触发 F2（文件详情）
- 文件图标根据扩展名区分：📄 txt/md、📕 json/js/ts/py、📊 csv、🖼️ png/jpg、📦 其他

**验收标准**：
- [ ] 应用启动后显示 "选择文件夹" 按钮
- [ ] 点击按钮弹出系统目录选择对话框（非文件选择）
- [ ] 选择目录后2秒内文件树渲染完成
- [ ] 文件夹可展开/收起，动画流畅
- [ ] 点击文件节点有明确选中态（高亮 + 右侧面板联动）
- [ ] 至少4种文件类型有不同的视觉图标
- [ ] 深度超过2层的子目录不显示（或显示 "更多..." 提示）
- [ ] 超过100个文件时截断并提示 "仅展示前100个条目"

---

### F2：文件详情面板
**描述**：右侧面板顶部展示当前选中文件的基本信息和内容预览（文本文件）。

**具体行为**：
- 文件被选中时展示：文件名、类型、大小（格式化如 "2.3 MB"）、修改时间
- 文本文件（.txt .md .json .js .ts .py .csv）展示前200行内容预览，超出部分截断并提示 "仅展示前200行"
- 非文本文件展示类型说明："该文件类型暂不支持内容预览"
- 未选中文件时展示空状态提示："请先选择一个文件夹以浏览文件"
- 详情区域固定高度，文件内容区域可独立滚动

**验收标准**：
- [ ] 未选中文件时展示空状态
- [ ] 选中文件后详情信息在200ms内更新
- [ ] 文件大小以人类可读格式展示（B/KB/MB/GB自动换算）
- [ ] 文本文件显示内容预览（前200行），带滚动条
- [ ] 非文本文件显示 "暂不支持内容预览" 提示
- [ ] 内容预览区域文字等宽字体，可读性好

---

### F3：AI文件分析
**描述**：对选中的文件调用AI进行智能分析，生成文件摘要和关键信息提取。

**具体行为**：
- 文件选中后，右侧面板出现「AI分析」按钮
- 点击按钮 → 显示 loading 状态（骨架屏或 spinner）
- 分析完成后展示：
  - **文件摘要**：200字以内的内容概括
  - **关键信息**：3-5条要点提取（bullet list）
  - **分析时间**：标注 "AI分析耗时 Xs"
- 同一文件重复点击 → 缓存结果，不调用API
- API调用失败 → 自动降级为Mock响应（结构与真实返回一致）

**验收标准**：
- [ ] 点击按钮后 loading 状态立即出现（< 100ms）
- [ ] 真实API成功时展示分析结果
- [ ] 真实API失败时展示Mock结果，用户无感知断裂
- [ ] 失败时有非侵入式错误提示（toast或inline message）
- [ ] 重复分析同一文件不重复请求

---

### F4：AI对话问答
**描述**：在文件分析结果下方提供对话输入框，用户可对当前文件内容进行多轮提问。

**具体行为**：
- 底部固定对话输入框（placeholder: "关于这个文件，你想问什么？"）
- 发送消息后：
  - 用户消息气泡立即展示（右侧，蓝色）
  - AI响应以流式输出展示（左侧，灰色），打字机效果
  - 消息按时间顺序排列，可滚动
- 对话上下文限定为当前选中文件的内容（system prompt 注入文件摘要）
- 切换文件 → 清空当前对话，新文件开启新对话
- 输入为空时发送按钮禁用

**验收标准**：
- [ ] 发送消息后用户气泡立即出现
- [ ] AI响应以流式/分段方式展示（非整段一次性出现）
- [ ] 切换文件后对话内容清空
- [ ] 空输入时发送按钮禁用
- [ ] API失败时展示友好错误提示，不影响已有对话历史

---

### F5：应用框架（Electron壳）
**描述**：Electron主进程搭建，窗口管理、IPC通信、环境变量读取。

**具体行为**：
- 主进程读取 `.env` 中的千问API Key（`DASHSCOPE_API_KEY`）
- 通过 IPC 安全传递 API Key 到渲染进程（不暴露到 preload 之外的全局变量）
- 窗口尺寸：1200×800，最小尺寸 900×600
- 自定义标题栏（非系统默认），包含应用名称和关闭/最小化按钮
- 开发模式支持热重载（Vite HMR + electron-reload）

**验收标准**：
- [ ] `npm start` 启动后窗口正常显示，无白屏
- [ ] `.env` 中配置API Key后应用可读取
- [ ] `npm run build` 成功打包为可执行文件
- [ ] 打包后的应用在至少一个平台（Win或macOS）可运行

## 2. 明确排除项（Out-of-Scope）

| 排除项 | 理由 |
|--------|------|
| **用户登录/认证** | 与核心展示链路无关。登录流程会引入OAuth/Token管理复杂度，且Demo场景下不需要真实用户 |
| **实时文件同步** | 需要WebSocket或轮询机制，增加后端依赖。Mock文件树已足够展示文件浏览功能 |
| **数据库持久化** | Mini Demo无需持久化层。对话历史和分析结果保存在React状态即可 |
| **拖拽文件上传** | 边缘功能，不影响核心链路（浏览→分析→问答）。如需展示上传能力，Mock文件树已覆盖 |
| **多平台打包（Linux/Win/macOS全打）** | 面试场景下只需保证一个平台可运行。Electron跨平台打包配置差异大，全打会引入不必要的不确定性 |
| **文件内容真实预览（PDF渲染、图片查看）** | PDF.js等库体积大、集成复杂。用文件元数据 + AI分析已足够传达"知识助手"价值 |
| **多文件批量分析** | 核心链路是单文件深度分析。批量是增量功能，不影响评审对Demo完整性的判断 |

## 4. 非功能需求

### 4.1 性能
- 首屏加载（文件树渲染）：< 500ms
- 文件选中到详情更新：< 200ms
- AI分析请求发出到首字节（TTFB）：取决于API响应，但UI loading态需在 < 100ms 出现
- 对话消息发送后，AI首字响应 < 3s（网络正常情况）

### 4.2 产物体积
- 开发环境：不限制
- 打包产物：**< 200MB**（Electron + Chromium基线约120MB，应用代码+依赖控制在80MB以内）
- 安装包（electron-builder）：**< 100MB**（asar压缩后）

### 4.3 错误处理策略
- **三层降级**：真实API → 格式兼容Mock → 静态兜底文案
- 所有网络请求必须有 timeout（15s）和 retry（最多1次）
- 错误信息不能暴露API细节（如URL、Key、HTTP状态码），统一展示为"服务暂时不可用，已切换至本地模式"
- React Error Boundary 覆盖整个应用，崩溃时展示恢复按钮

### 4.4 离线可用性
- 应用不依赖持续在线状态：文件树为内置Mock数据，可离线浏览
- AI功能需要网络，但失败时自动降级为Mock响应（内置3-5个预设分析结果）
- 不强制联网检测，无"必须在线才能使用"的阻断

## 5. 外部依赖

### 5.1 千问API（DashScope）
- **服务**：阿里云 DashScope（百炼平台），模型 `qwen-turbo` 或 `qwen-plus`
- **端点**：`https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`（OpenAI兼容格式）
- **认证**：Bearer Token，通过环境变量 `DASHSCOPE_API_KEY` 注入
- **免费额度**：新用户有免费Token额度，足够Demo开发和评审使用
- **流式输出**：支持 `stream: true`，SSE格式返回

### 5.2 Key管理
- **开发环境**：项目根目录 `.env` 文件（已加入 `.gitignore`）
- **环境变量模板**：提供 `.env.example` 提交到代码仓库，标注所需变量名
- **Electron安全**：API Key 仅在主进程读取，通过 IPC 按需传递，不暴露到 `window` 全局
- **禁止**：任何代码路径将 Key 打印到 console 或日志文件

### 5.3 API降级策略
| 故障场景 | 降级行为 |
|----------|----------|
| API Key未配置或无效 | 启动时 toast 提示"未检测到API Key，已切换至演示模式"，全自动Mock |
| API超时（> 15s） | 第1次自动重试，仍失败则展示Mock响应 + toast提示 |
| API返回非200（配额耗尽等） | 同超时降级，Mock响应 |
| 完全断网 | 同超时降级，Mock响应 |
| Mock数据也未加载（极端） | 展示静态兜底文案："文件分析功能暂时不可用" |
